  <action>
Add CoinbaseDownloader class to the bottom of downloader.py (after BybitDownloader), following the same interface pattern.

**Class constants:**
```python
REST_BASE: str = "https://api.coinbase.com/api/v3/brokerage/products"
WS_URL: str = "wss://advanced-trade-ws.coinbase.com"
```

**Constructor:** Same signature as BybitDownloader but default `symbol="BTC-USD"`, `depth=10`, `output_dir="data/"`.

**fetch_snapshot():**
- GET `{REST_BASE}/{self.symbol}/book?level=2` via `_retry_get()`
- Response JSON has top-level `bids` and `asks`, each a list of `[price_str, size_str, num_orders_int]`
- Take only top `self.depth` entries (they come pre-sorted: bids desc, asks asc)
- Raise `ValueError` if fewer than `self.depth` levels returned
- Build row dict using same schema constants (BID_PRICE_COLS, ASK_PRICE_COLS, etc.)
- Timestamp: response has `"time"` field as ISO string (e.g. `"2026-03-22T10:30:45.123456Z"`) — convert to microseconds since epoch using `datetime.fromisoformat(time_str.replace("Z","+00:00")).timestamp() * 1_000_000`
- Trade fields: TRADE_PRICE=NaN, TRADE_SIZE=NaN, TRADE_SIDE=0 (REST orderbook has no trades)
- Return single-row DataFrame with columns=ALL_COLUMNS

**record_websocket():** Sync wrapper identical to BybitDownloader's — `asyncio.run(self._ws_record(...))`.

**_ws_record() — Coinbase-specific WebSocket logic:**
- Subscribe message: `{"type": "subscribe", "product_ids": [self.symbol], "channel": "level2"}`
- Message structure differs from Bybit:
  - Top-level `"channel"` == `"l2_data"` for data messages (vs heartbeats)
  - `"events"` is a list; each event has `"type"` (`"snapshot"` or `"update"`) and `"updates"` list
  - Each update entry: `{"price_level": "45000.50", "new_quantity": "1.5", "side": "bid"}` or `"side": "offer"` (Coinbase uses "offer" for asks)
  - `new_quantity == "0"` means remove the level
- Timestamp: use `event.get("event_time", "")` from the parent event object — ISO string, convert same as above. Fall back to `time.time() * 1_000_000` if missing.
- Book state: same `bid_book: dict[str, float]`, `ask_book: dict[str, float]` pattern
- For snapshot event: clear both books, apply all updates
- For update event: apply updates (delete if new_quantity == "0", else set)
- Reuse `_book_to_row()` and `_flush_buffer()` — these are identical to Bybit's. Either copy them into CoinbaseDownloader or extract as module-level helpers.

**_book_to_row():** Identical logic to BybitDownloader's but timestamp input is already in microseconds (not milliseconds), so no `* 1_000` conversion needed — set `row[TIMESTAMP] = ts_us` directly.

**Important pitfalls:**
- Coinbase uses `"offer"` (not `"ask"`) for the ask side in level2 updates — handle both: `if side in ("bid",): ask_book else bid_book` ... actually: `if side == "bid": bid_book else ask_book` covers "offer"
- The WebSocket may send heartbeat messages with `"channel": "heartbeats"` — skip any message where `channel != "l2_data"`
- Some messages have `"events": []` (empty) — skip gracefully
- The initial snapshot can be large (many price levels) — the `_book_to_row()` then extracts top depth

**Do NOT** require any API key, JWT, or authentication — Coinbase level2 is public.
