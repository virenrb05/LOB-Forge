  <name>Task 1: Implement cosine noise schedule</name>
  <files>lob_forge/generator/noise_schedule.py</files>
  <action>
    Replace the stub with a full CosineNoiseSchedule (nn.Module) implementing Nichol & Dhariwal 2021 cosine schedule.

    Constructor args: `num_timesteps: int = 1000, s: float = 0.008` (s is the small offset preventing beta from being too small near t=0).

    Compute and register as buffers:
    - `betas`: β_t = 1 - (ᾱ_t / ᾱ_{t-1}), clipped to [0, 0.999]
    - `alphas`: α_t = 1 - β_t
    - `alphas_cumprod`: ᾱ_t (cumulative product of alphas) — this is the key schedule
    - `alphas_cumprod_prev`: ᾱ_{t-1} (shifted by 1, with ᾱ_0 = 1.0)
    - `sqrt_alphas_cumprod`: √ᾱ_t
    - `sqrt_one_minus_alphas_cumprod`: √(1 - ᾱ_t)
    - `posterior_mean_coef1`: √ᾱ_{t-1} * β_t / (1 - ᾱ_t)
    - `posterior_mean_coef2`: √α_t * (1 - ᾱ_{t-1}) / (1 - ᾱ_t)
    - `posterior_variance`: β_t * (1 - ᾱ_{t-1}) / (1 - ᾱ_t)
    - `posterior_log_variance_clipped`: log(posterior_variance) clipped to avoid log(0)

    Cosine schedule formula: f(t) = cos((t/T + s) / (1 + s) * π/2)², then ᾱ_t = f(t) / f(0).

    Methods:
    - `q_sample(x_0: Tensor, t: Tensor, noise: Tensor | None = None) -> Tensor`: Forward diffusion — returns x_t = √ᾱ_t * x_0 + √(1-ᾱ_t) * ε. Use `_extract(tensor, t, x_shape)` helper to gather coefficients at timestep t and reshape for broadcasting.
    - `_extract(a: Tensor, t: Tensor, x_shape: tuple) -> Tensor`: Index into schedule tensor `a` at positions `t`, reshape to (B, 1, 1, ...) for broadcasting with x_shape.

    All tensors should be float32. Use `register_buffer` so they move with `.to(device)`.
  </action>
  <verify>
    python -c "
from lob_forge.generator.noise_schedule import CosineNoiseSchedule
import torch
s = CosineNoiseSchedule(1000)
# alpha_bar monotonically decreasing
assert (s.alphas_cumprod[1:] < s.alphas_cumprod[:-1]).all()
# alpha_bar starts near 1, ends near 0

assert x_t.shape == (2, 100, 40)
print('noise_schedule OK')

<task type="auto">
  <name>Task 2: Implement EMA wrapper</name>
  <files>lob_forge/generator/ema.py</files>
  <action>
    Create ExponentialMovingAverage class that maintains a shadow copy of model parameters.

    Constructor: `__init__(self, model: nn.Module, decay: float = 0.9999)`
    - Store `decay` as attribute
    - Create shadow parameters as a dict mapping parameter name → detached clone
    - Use `model.named_parameters()` to iterate

    Methods:
    - `update(model: nn.Module) -> None`: For each parameter, shadow = decay * shadow + (1 - decay) * param. Use `torch.no_grad()`.
    - `apply_shadow(model: nn.Module) -> None`: Copy shadow params into model (for inference).
    - `restore(model: nn.Module) -> None`: Copy original params back (after inference). Store originals in `apply_shadow` before overwriting.
    - `state_dict() -> dict`: Return shadow params dict for saving.
    - `load_state_dict(state_dict: dict) -> None`: Load shadow params from checkpoint.

    Keep it simple — no registration as nn.Module, just a plain class. Store shadow params on same device as original params.
  </action>
  <verify>
    python -c "
import torch, torch.nn as nn
from lob_forge.generator.ema import ExponentialMovingAverage
model = nn.Linear(10, 5)
ema = ExponentialMovingAverage(model, decay=0.99)
# Initial shadow should equal model params
for name, p in model.named_parameters():
    assert torch.allclose(ema.shadow_params[name], p.data)
# After update with changed params, shadow should differ
with torch.no_grad():
    for p in model.parameters():
        p.add_(1.0)
ema.update(model)
# Shadow should have moved toward new params
for name, p in model.named_parameters():
    assert not torch.allclose(ema.shadow_params[name], p.data)
# state_dict roundtrip
sd = ema.state_dict()
ema2 = ExponentialMovingAverage(model, decay=0.99)
ema2.load_state_dict(sd)
for k in sd:
    assert torch.allclose(ema.shadow_params[k], ema2.shadow_params[k])
print('EMA OK')
"
  </verify>
  <done>EMA tracks shadow params, apply/restore works, state_dict serialization works</done>
