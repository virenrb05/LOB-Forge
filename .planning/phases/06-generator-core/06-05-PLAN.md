    **3. Implement lob_forge/generator/train.py:**

    `train_generator(cfg: DictConfig) -> Path` function:
    - Build DiffusionModel from cfg.generator params
    - Create EMA wrapper with cfg.generator.ema_decay
    - Create LOBSequenceDataset from preprocessed data (cfg.data.data_dir / "preprocessed.parquet")
      - Pass explicit book feature columns (40 columns) to avoid derived features
      - Use cfg.data.sequence_length
    - Create DataLoader with cfg.generator.training.batch_size, pin_memory=False for MPS
    - Detect device: MPS > CUDA > CPU (matching predictor trainer pattern)
    - Optimizer: AdamW with cfg.generator.optimizer.lr, weight_decay
    - Training loop:
      - For each epoch:
        - For each batch (sequence, regime):
          - loss = model.training_loss(sequence, regime)
          - loss.backward(), optimizer.step(), optimizer.zero_grad()
          - ema.update(model)
        - Log epoch loss to wandb (if enabled)
        - Every save_every epochs: save checkpoint (model state_dict, ema state_dict, optimizer state_dict, epoch)
        - Every sample_every epochs: generate a small sample batch and log shapes to wandb
    - Return path to best/last checkpoint

    Follow the predictor trainer.py patterns for:
    - Device detection (_get_device helper)
    - wandb integration (conditional on cfg.wandb.enabled)
    - Checkpoint saving format
    - Using `_get()` helper for OmegaConf/dict access if needed

    Keep it simple — no early stopping for diffusion (loss doesn't plateau the same way). Just run for configured epochs.
