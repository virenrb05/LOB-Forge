Implement `ExecutionResult` dataclass and 4 baseline classes in `lob_forge/executor/baselines.py`.

**ExecutionResult dataclass:**
```python
@dataclass
class ExecutionResult:
    episode_cost: float         # total cost paid
    implementation_shortfall: float  # (exec_vwap - arrival_price) * inventory
    remaining_inventory: float  # leftover at episode end (0 = fully executed)
    n_steps: int                # steps taken before termination/truncation
    actions: list[int]          # sequence of actions taken
```
`implementation_shortfall` measures execution quality: positive = bought at prices above arrival mid, negative = better than arrival.
`arrival_price` = mid-price at episode start (from first obs: (col_0 + col_20) / 2).

**Base class:**
```python
class BaselineAgent:
    def select_action(self, obs: np.ndarray, remaining: float, step: int) -> int:
        raise NotImplementedError

    def run_episode(self, env: LOBExecutionEnv, seed: int = 0) -> ExecutionResult:
        obs, info = env.reset(seed=seed)
        # arrival price from first observation
        arrival_price = (obs[-1, 0] + obs[-1, 20]) / 2.0  # last row = current step
        actions = []
        step = 0
        while True:
            action = self.select_action(obs, info.get("remaining", env.inventory), step)
            obs, reward, terminated, truncated, info = env.step(action)
            actions.append(action)
            step += 1
            if terminated or truncated:
                break
        exec_vwap_approx = arrival_price + (info["episode_cost"] / max(env.inventory - info["remaining"], 1e-9))
        is_shortfall = (exec_vwap_approx - arrival_price) * env.inventory
        return ExecutionResult(
            episode_cost=info["episode_cost"],
            implementation_shortfall=is_shortfall,
            remaining_inventory=info["remaining"],
            n_steps=step,
            actions=actions,
        )
```

**TWAPBaseline:**
- TWAP divides inventory evenly over horizon steps: target_per_step = inventory / horizon
- select_action: compute how much has been executed so far, pick MARKET_SMALL (action 1) to make progress. Always return action 1 (MARKET_SMALL, 1% of remaining) — simplest TWAP approximation. Return 0 (WAIT) only when remaining <= 0.

**VWAPBaseline:**
- VWAP approximation: proportional to volume proxy per step
- Volume proxy: use a simple sinusoidal pattern with random noise to simulate intraday volume variation (no real volume data available since data is z-score normalized)
- `volume_weights[t] = max(0.01, sin(pi * t / horizon) + 0.1 * rng.random())` — U-shaped intraday volume
- Normalize: target_fraction[t] = volume_weights[t] / sum(volume_weights[:horizon])
- select_action: if remaining > 0 and this step's target says to execute, return action 1 (MARKET_SMALL); else WAIT
- Precompute volume schedule in __init__ with a seeded rng (seed=42 for reproducibility)

**AlmgrenChrissBaseline:**
- Almgren-Chriss optimal liquidation: trades off market impact vs timing risk
- Simplified closed-form: deterministic liquidation trajectory
- Parameters: eta=0.1 (impact), sigma=0.3 (vol estimate), lam=1e-5 (risk aversion)
- kappa = sqrt(lam * sigma^2 / eta)  [decay rate]
- n_j = inventory * sinh(kappa * (horizon - j)) / sinh(kappa * horizon)  [shares at step j]
- delta_j = n_j - n_{j+1}  [trade size at step j]
- select_action: compute delta_j for current step; if delta_j > 0.01 * inventory, return MARKET_MED (action 2); else WAIT
- Precompute full trajectory in reset_episode() (called at start of run_episode)
- Add reset_episode() method that takes inventory and horizon; run_episode() calls it before the loop

**RandomBaseline:**
- select_action: return env.action_space.sample() — uniform random from [0, 6]
- Use env's RNG (indirectly via gymnasium seeding) — no explicit rng needed
