<feature>
  <name>DuelingDQN + PrioritizedReplayBuffer</name>
  <files>lob_forge/executor/agent.py, tests/test_dqn_agent.py</files>
  <behavior>
    **DuelingDQN(obs_shape, n_actions, hidden_dim=256):**
    - obs_shape: tuple e.g. (100, 40) — seq_len × features
    - Input is flattened: (B, seq_len*40)
    - Shared trunk: Linear(seq_len*40, hidden_dim) → ReLU → Linear(hidden_dim, hidden_dim) → ReLU
    - Value stream: Linear(hidden_dim, 1) → scalar V(s)
    - Advantage stream: Linear(hidden_dim, n_actions) → A(s,a)
    - Q(s,a) = V(s) + A(s,a) - mean(A(s,·))  [Wang et al. 2016 dueling formula]
    - forward(obs: Tensor[B, seq_len, 40]) → Q: Tensor[B, n_actions]
    - Input obs can be 3D (B, seq_len, 40); flatten to 2D inside forward

    **PrioritizedReplayBuffer(capacity, alpha=0.6, beta_start=0.4, beta_end=1.0, beta_steps=100_000):**
    - Stores (obs, action, reward, next_obs, done) transitions
    - Priority-based sampling: P(i) ∝ priority_i^alpha
    - IS correction weights: w_i = (N * P(i))^(-beta) / max_w
    - beta anneals linearly from beta_start to beta_end over beta_steps calls to sample()
    - push(obs, action, reward, next_obs, done): stores transition with max current priority
    - sample(batch_size) → (obs, actions, rewards, next_obs, dones, weights, indices)
      - obs, next_obs: torch.FloatTensor shape (batch, seq_len, 40)
      - actions: torch.LongTensor shape (batch,)
      - rewards, dones: torch.FloatTensor shape (batch,)
      - weights: torch.FloatTensor shape (batch,) — IS correction weights, clamped [1e-8, 1]
      - indices: list[int] — for update_priorities()
    - update_priorities(indices, td_errors): updates stored priorities = |td_error| + 1e-6

    Use a simple list (deque with maxlen=capacity) for storage. No segment tree needed — use numpy random choice with probabilities for sampling (acceptable for 100k buffer).

    Cases:
    - DuelingDQN forward: obs (4, 100, 40) → Q (4, 7)
    - Buffer at capacity: oldest transition evicted on push
    - Sample before buffer has batch_size entries: raise ValueError
    - update_priorities with batch_size indices updates only those indices
  </behavior>

    No external RL libraries — implement from scratch per EXEC-03.
  </implementation>
</feature>
