**Function signature:**
```python
def train_agent(cfg) -> Path:
    """Train DQN agent via 3-stage curriculum. Returns path to final checkpoint."""
```

**Hyperparameters from cfg (all under cfg.executor):**
- `seq_len`, `inventory`, `horizon`, `order_sizes`, `limit_offsets_bps`
- `n_actions` (7), `gamma` (0.99), `lr` (5e-4)
- `buffer_size` (100_000), `batch_size` (64)
- `epsilon_start` (1.0), `epsilon_end` (0.01), `epsilon_decay` (0.995)
- `target_update_freq` (1000)
- `curriculum_stages`: ["low_vol", "mixed", "adversarial"]

**Device selection:** Use MPS if available (`torch.backends.mps.is_available()`), else CUDA if available, else CPU.

**Curriculum stage definitions:**
```python
STAGE_CONFIG = {
    "low_vol":    {"regime": 0, "steps": 50_000, "mode": "real"},
    "mixed":      {"regime": 1, "steps": 75_000, "mode": "real"},
    "adversarial":{"regime": 2, "steps": 50_000, "mode": "real"},
}
```
Use `mode="real"` for all stages (synthetic mode requires loaded DiffusionModel — Phase 9 uses real data for training speed). The regime label is logged but doesn't change env behavior in real mode — regime variation comes from different LOB data segments.

**Training loop per stage:**
1. Instantiate fresh `LOBExecutionEnv` from `cfg` with a loaded Parquet file OR random dummy data.
   - Check `cfg.executor.get("data_path", None)`. If None, generate `np.random.randn(10_000, 40).astype(np.float32)` as dummy data (allows training without real data present). Log a warning if using dummy data.
   - Pass: `lob_data=data, seq_len=cfg.executor.seq_len, inventory=cfg.executor.inventory, horizon=cfg.executor.horizon`
2. Create or load DuelingDQN (online net and target net — target is copy of online)
3. Optimizer: `torch.optim.Adam(online_net.parameters(), lr=cfg.executor.lr)`
4. Replay buffer: `PrioritizedReplayBuffer(capacity=cfg.executor.buffer_size)`
5. Run episodes until stage_steps reached:
   - epsilon-greedy action selection: if `random() < epsilon`: random; else argmax(online_net(obs))
   - Step env, push transition to buffer
   - If buffer has >= batch_size samples: sample batch, compute Double-DQN loss, update
   - Double-DQN update:
     ```python
     with torch.no_grad():
         next_actions = online_net(next_obs).argmax(dim=1)  # online selects
         next_q = target_net(next_obs).gather(1, next_actions.unsqueeze(1)).squeeze(1)  # target evaluates
         targets = rewards + gamma * next_q * (1 - dones)
     current_q = online_net(obs).gather(1, actions.unsqueeze(1)).squeeze(1)
     td_errors = (targets - current_q).detach().abs()
     loss = (weights * F.mse_loss(current_q, targets, reduction='none')).mean()
     ```
   - Backprop loss, optimizer step, update priorities
   - Hard-copy target net every `target_update_freq` steps: `target_net.load_state_dict(online_net.state_dict())`
   - Decay epsilon: `epsilon = max(epsilon_end, epsilon * epsilon_decay)`
   - Log every 1000 steps: step, epsilon, mean loss (last 100), mean episode reward (last 10 episodes)
   - On episode end: reset env
6. Save stage checkpoint after each stage completes

**Checkpoint format:**
```python
checkpoint = {
    "stage": stage_name,
    "online_net": online_net.state_dict(),
    "target_net": target_net.state_dict(),
    "optimizer": optimizer.state_dict(),
    "epsilon": epsilon,
    "step": total_steps,
}
torch.save(checkpoint, ckpt_path)
```
Save to: `checkpoints/executor_{stage_name}.pt` (create dir if not exists).

**Stage continuation:** Each stage starts from previous stage's checkpoint if it exists. Load online_net, target_net, optimizer state, epsilon before stage 2 and stage 3.

**Return:** Path to final stage checkpoint (stage 3).

**Error guard:** If loss is NaN at any step, log error and break out of training loop (don't crash). Log "NaN loss detected at step {step}, stopping early."

Do NOT import wandb — logging goes to stdout via print(). wandb integration is Phase 10 concern.
