<feature>
  <name>Temporal Splits with Purge Gaps</name>
  <files>lob_forge/data/splits.py, tests/test_splits.py</files>
  <behavior>
    temporal_split(n_rows, ratios=(0.7, 0.15, 0.15), purge_gap=0)
      → tuple[np.ndarray, np.ndarray, np.ndarray]

    Algorithm:
    1. train_end = int(n_rows * ratios[0])
    2. val_start = train_end + purge_gap
    3. val_end = val_start + int(n_rows * ratios[1])
    4. test_start = val_end + purge_gap
    5. test_end = n_rows
    6. Return (arange(0, train_end), arange(val_start, val_end), arange(test_start, test_end))

    Test cases:
    - n_rows=1000, purge_gap=0: train=[0..699], val=[700..849], test=[850..999]
    - n_rows=1000, purge_gap=10: train=[0..699], val=[710..859], test=[870..999]
    - No overlap: train ∩ val = ∅, val ∩ test = ∅, train ∩ test = ∅
    - Ordering: max(train) < min(val) < max(val) < min(test)
    - Gap: min(val) - max(train) >= purge_gap + 1
    - Gap: min(test) - max(val) >= purge_gap + 1
    - Total rows used <= n_rows (some rows lost to purge gaps)
    - Edge case: purge_gap larger than val segment → val becomes empty (return empty array)
    - Edge case: n_rows=10, ratios=(0.7,0.15,0.15), purge_gap=0 → train=[0..6], val=[7..8], test=[9]
  </behavior>
  <implementation>
    Pure numpy function. No pandas dependency needed.

    Use np.arange to create index arrays. Handle edge cases where purge_gap causes segments to overlap or become empty — return empty np.array([]) for segments that don't fit.

    Type hints: return tuple[np.ndarray, np.ndarray, np.ndarray] with dtype int64.
  </implementation>
</feature>
