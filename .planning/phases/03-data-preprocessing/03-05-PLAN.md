  <name>Task 1: Implement LOBDataset and LOBSequenceDataset</name>
  <files>lob_forge/data/dataset.py</files>
  <action>
    Replace the stub dataset.py with two PyTorch Dataset classes:

    1. LOBDataset(torch.utils.data.Dataset):
       - __init__(self, parquet_path, sequence_length=100, horizons=[10, 20, 50, 100], feature_cols=None):
         - Read Parquet file via PyArrow (pa.parquet.read_table with memory_map=True for efficiency)
         - Convert to numpy arrays for features and labels
         - feature_cols: list of feature column names to use (if None, auto-detect: all float64 columns except label_*, timestamp, trade_side)
         - Store as numpy arrays (convert to tensors in __getitem__ for memory efficiency)
       - __len__: total_rows - sequence_length (sliding window)
       - __getitem__(idx) → tuple[torch.Tensor, torch.Tensor]:
         - features: shape (sequence_length, num_features) float32
         - labels: shape (num_horizons,) int64 — label values at row idx + sequence_length (the prediction target)
         - Labels are the values at the END of the window (what we predict after seeing the sequence)

    2. LOBSequenceDataset(torch.utils.data.Dataset):
       - __init__(self, parquet_path, sequence_length=100, feature_cols=None, vol_quantiles=(0.33, 0.67)):
         - Same Parquet loading as LOBDataset
         - Precompute regime labels from realized volatility:
           - For each possible window [i : i + sequence_length], compute realized vol = std(mid_returns) over the window
           - Classify into regime: realized_vol < q33 → 0 (low-vol), q33..q67 → 1 (normal), > q67 → 2 (high-vol)
           - Quantile thresholds computed from ALL windows in the dataset
       - __len__: total_rows - sequence_length
       - __getitem__(idx) → tuple[torch.Tensor, torch.Tensor]:
         - sequence: shape (sequence_length, num_features) float32
         - regime: shape () int64 (scalar regime label)

    Both classes should:
    - Use float32 for feature tensors (MPS compatibility — float16 not supported on all ops)
    - Use int64 for label/regime tensors
    - NOT load the entire dataset into GPU memory — create tensors in __getitem__

    For mid_returns in regime computation: if mid_return_1 column exists, use it. Otherwise compute from mid_price.
  </action>
  <verify>
    python -c "
    import torch, numpy as np, pandas as pd, pyarrow as pa, pyarrow.parquet as pq, tempfile, os
    # Create minimal test parquet
    n = 500
    data = {
        'timestamp': np.arange(n, dtype=np.int64),
        'mid_price': 100.0 + np.cumsum(np.random.randn(n) * 0.001),
        'spread': np.full(n, 0.01),
    }
    for i in range(1, 11):
        data[f'bid_price_{i}'] = data['mid_price'] - 0.005 * i
        data[f'bid_size_{i}'] = np.random.rand(n) * 100
        data[f'ask_price_{i}'] = data['mid_price'] + 0.005 * i
        data[f'ask_size_{i}'] = np.random.rand(n) * 100
    data['trade_price'] = data['mid_price']
    data['trade_size'] = np.random.rand(n) * 10
    data['trade_side'] = np.random.choice([-1, 0, 1], n).astype(np.int8)
    data['mid_return_1'] = np.concatenate([[np.nan], np.diff(data['mid_price']) / data['mid_price'][:-1]])
    data['label_h10'] = np.random.choice([0.0, 1.0, 2.0], n)
    df = pd.DataFrame(data)
    tmp = tempfile.mktemp(suffix='.parquet')
    pq.write_table(pa.Table.from_pandas(df), tmp)
    from lob_forge.data.dataset import LOBDataset, LOBSequenceDataset
    ds1 = LOBDataset(tmp, sequence_length=50, horizons=[10])
    feat, lab = ds1[0]
    print(f'LOBDataset: features {feat.shape}, labels {lab.shape}, dtype {feat.dtype}')
    ds2 = LOBSequenceDataset(tmp, sequence_length=50)
    seq, reg = ds2[0]
    print(f'LOBSequenceDataset: sequence {seq.shape}, regime {reg.shape}, dtype {seq.dtype}')
    os.unlink(tmp)
    print('OK')
