  <name>Task 1: Implement compute_ofi() and compute_mlofi()</name>
  <files>lob_forge/data/features.py</files>
  <action>
    Add two new functions to `features.py`:

    **compute_ofi(df) -> pd.Series:**
    Level-1 Order Flow Imbalance measuring *changes* in book quantities between consecutive snapshots.
    Formula (Cont, Kukanov & Stoikov 2014):
    ```
    delta_bid_size = bid_size_1 - bid_size_1.shift(1)
    delta_ask_size = ask_size_1 - ask_size_1.shift(1)
    OFI_t = delta_bid_size * I(bid_price_1 >= bid_price_1.shift(1))
          - delta_ask_size * I(ask_price_1 <= ask_price_1.shift(1))
    ```
    Where I() is the indicator function. First row is NaN (no previous snapshot).
    Return a pd.Series named "ofi".

    **compute_mlofi(df, levels=10, decay=0.5) -> pd.Series:**
    Multi-Level OFI extending OFI across multiple book levels with exponential decay weighting.
    For each level i (1..levels):
    ```
    ofi_i = delta(bid_size_i) * I(bid_price_i >= prev_bid_price_i)
          - delta(ask_size_i) * I(ask_price_i <= prev_ask_price_i)
    ```
    MLOFI = sum(decay^(i-1) * ofi_i for i in 1..levels)
    First row is NaN. Return a pd.Series named "mlofi".

    Use the existing `_EPS` constant pattern. Use vectorized pandas operations (no loops over rows).
    The level loop (10 iterations) is fine — it's the row loop that must be vectorized.

    **Also update `compute_all_features()`:**
    Add `ofi` and `mlofi` columns to the output. Update the docstring to reflect the new column count (18 + 2 = 20 columns).

    **Do NOT remove** `compute_order_imbalance()` or `compute_depth_imbalance()` — they are valid static features used elsewhere. OFI/MLOFI are *additional* flow-based features.
  </action>
