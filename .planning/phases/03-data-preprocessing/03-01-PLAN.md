<feature>
  <name>LOB Derived Features (6 functions)</name>
  <files>lob_forge/data/features.py, tests/test_features.py</files>
  <behavior>
    All functions accept a pd.DataFrame with unified LOB schema columns.

    1. compute_mid_returns(df, lookbacks=[1, 5, 10, 50]) → DataFrame
       - For each K in lookbacks, add column `mid_return_K` = (mid_price - mid_price.shift(K)) / mid_price.shift(K)
       - First K rows of each return column are NaN
       - Test: prices [100, 101, 102, 103, 104] → mid_return_1 at row 1 = 0.01

    2. compute_order_imbalance(df) → Series
       - Formula: (bid_size_1 - ask_size_1) / (bid_size_1 + ask_size_1)
       - Range: [-1, 1]. When bid_size_1 == ask_size_1 → 0.0
       - Test: bid_size_1=100, ask_size_1=50 → (100-50)/(100+50) = 0.3333

    3. compute_microprice(df) → Series
       - Formula: (bid_price_1 * ask_size_1 + ask_price_1 * bid_size_1) / (bid_size_1 + ask_size_1)
       - When bid_size_1 == ask_size_1, equals mid_price exactly
       - Test: bid=100 ask=101, sizes equal → 100.5; bid=100 ask=101, bid_size=200 ask_size=100 → (100*100 + 101*200)/(200+100) = 100.6667

    4. compute_depth_imbalance(df, levels=10) → DataFrame
       - For each level i=1..10: `depth_imb_i` = (bid_size_i - ask_size_i) / (bid_size_i + ask_size_i)
       - Each column range [-1, 1]
       - Test: all bid_sizes > ask_sizes → all positive

    5. compute_spread_bps(df) → Series
       - Formula: spread / mid_price * 10000
       - Test: spread=0.01, mid=100 → 1.0 bps

    6. compute_vpin(df, n_buckets=50) → Series
       - Bulk volume classification: for each trade, buy_vol = trade_size * Φ(ln(P_t/P_{t-1}) / σ), sell_vol = trade_size - buy_vol
       - Where Φ is standard normal CDF, σ is rolling std of log returns over 50 trades
       - Aggregate into equal-volume buckets, VPIN = mean(|buy_vol - sell_vol| / bucket_vol) over n_buckets
       - For rows without trades (trade_size is NaN), forward-fill the last VPIN value
       - Range: [0, 1]. 0 = balanced, 1 = all informed
       - Test: all trades same direction → VPIN near 1.0; alternating buy/sell equal size → VPIN near 0.0

    7. compute_all_features(df, lookbacks=[1, 5, 10, 50]) → DataFrame
       - Calls all 6 functions, returns original df augmented with all derived columns
       - New columns: mid_return_1, mid_return_5, mid_return_10, mid_return_50, order_imbalance, microprice, depth_imb_1..depth_imb_10, spread_bps, vpin
       - Total new columns: 4 + 1 + 1 + 10 + 1 + 1 = 18
  </behavior>
  <implementation>
    Import column constants from schema.py (BID_PRICE_COLS, BID_SIZE_COLS, ASK_PRICE_COLS, ASK_SIZE_COLS, MID_PRICE, SPREAD, TRADE_PRICE, TRADE_SIZE).

    All functions are vectorized pandas operations — no row-level loops.

    For VPIN: use scipy.stats.norm.cdf for bulk volume classification. Handle NaN trade rows by skipping them in the rolling computation, then forward-fill the VPIN series. If scipy is not yet a dependency, add it to pyproject.toml (it's already a transitive dependency of PyTorch).

    For compute_all_features: call each function and pd.concat the results. Return the augmented DataFrame.

    Guard against division by zero in imbalance/microprice with a small epsilon (1e-12) in denominator.
  </implementation>
</feature>
