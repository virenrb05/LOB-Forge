<feature>
  <name>Mid-price Movement Labels</name>
  <files>lob_forge/data/labels.py, tests/test_labels.py</files>
  <behavior>
    compute_labels(df, horizons=[10, 20, 50, 100], threshold=0.00002) → DataFrame

    For each horizon h:
    1. Compute smoothed future mid-price: mean of mid_price[t+1 : t+h+1]
    2. Compute return: (smoothed_future - mid_price[t]) / mid_price[t]
    3. Classify:
       - return > threshold  → 2 (UP)
       - return < -threshold → 0 (DOWN)
       - otherwise           → 1 (STATIONARY)
    4. Output column: `label_h{h}` (e.g., label_h10, label_h20, label_h50, label_h100)
    5. Last h rows of each column are NaN (no future data available)

    Test cases:
    - Monotonically increasing prices [100, 100.01, 100.02, ...] → all UP labels
    - Flat prices [100, 100, 100, ...] → all STATIONARY labels
    - Decreasing prices → all DOWN labels
    - Mixed: known trajectory → deterministic expected labels (manually computed)
    - Causality test: create df, compute labels, modify mid_price at row 50, recompute labels.
      Labels at rows 0..49 must NOT change. Labels at rows 50..50+max_horizon may change.
    - Last h rows of label_h{h} must be NaN
    - Label dtype is float64 (to support NaN) with values in {0.0, 1.0, 2.0, NaN}
  </behavior>
  <implementation>
    Import MID_PRICE from schema.py.

    Use pandas rolling/shift operations — NOT row-level loops:
    - smoothed_future = mid_price.shift(-1).rolling(window=h, min_periods=h).mean()
      Wait — this shifts backward then rolls forward, which is wrong. Instead:
    - For horizon h: smoothed_future[t] = mean(mid_price[t+1], mid_price[t+2], ..., mid_price[t+h])
    - Implement as: mid_price.iloc[::-1].rolling(window=h, min_periods=h).mean().iloc[::-1].shift(-1)
    - Or equivalently: mid_price.shift(-1).rolling(window=h, min_periods=h).mean() with appropriate alignment

    Actually, simplest correct approach:
    - smoothed = mid_price.shift(-h).rolling(window=h).mean()  — NO, this isn't right either.
    - Correct: for each t, average mid_price[t+1..t+h]. Use: (mid_price.rolling(h).sum().shift(-h) ) / h
    - Even simpler: mid_price.shift(-1) starts at t+1. Then .rolling(h).mean() gives mean of [t+1-h+1..t+1] = [t-h+2..t+1] which is wrong.

    Best approach: compute cumsum, then future_sum[t] = cumsum[t+h] - cumsum[t], then smoothed = future_sum / h.

    Use numpy for clarity:
    ```
    cs = np.concatenate([[0], np.cumsum(mid_price.values)])
    for h in horizons:
        future_mean = (cs[h+1:] - cs[1:-h]) / h  # shape: n-h
        # Pad with NaN at end
    ```

    Return DataFrame with label columns added to input df.
  </implementation>
