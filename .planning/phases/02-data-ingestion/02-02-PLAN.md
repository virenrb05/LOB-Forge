  <name>Task 1: Implement BybitDownloader with REST snapshot and historical archive download</name>
  <files>lob_forge/data/downloader.py, pyproject.toml</files>
  <action>
    Implement `BybitDownloader` class in `lob_forge/data/downloader.py`:

    **Constructor:**
    ```
    __init__(self, symbol='BTCUSDT', depth=10, output_dir='data/')
    ```
    - Store params, create output_dir if needed
    - Base REST URL: `https://api.bybit.com/v5/market/orderbook`
    - Base archive URL: `https://public.bybit.com/trading/`

    **Method 1: `fetch_snapshot() -> pd.DataFrame`**
    - GET `{base_url}?category=linear&symbol={symbol}&limit={depth*5}` (request 50 levels, take top depth)
    - Parse response JSON: `result.b` = bids (list of [price, size]), `result.a` = asks (list of [price, size])
    - Construct single-row DataFrame matching LOB_SCHEMA:
      - timestamp from `result.ts` (convert ms → μs)
      - mid_price = (best_bid + best_ask) / 2
      - spread = best_ask - best_bid
      - bid_price_1..{depth}, bid_size_1..{depth} from `result.b`
      - ask_price_1..{depth}, ask_size_1..{depth} from `result.a`
      - trade_price/trade_size/trade_side = NaN/NaN/0 (not available from orderbook endpoint)
    - Return DataFrame

    **Method 2: `download_historical(start_date: str, end_date: str) -> Path`**
    - Download CSV archives from `https://public.bybit.com/trading/{symbol}/`
    - Files are named `{symbol}{date}.csv.gz` for each date
    - These are TRADE files (not order books). Parse columns: timestamp, symbol, side, size, price, ...
    - Reconstruct order book from trade stream:
      - Maintain a local book state (sorted bids/asks)
      - For each trade, update the book: remove filled levels, add new levels
      - Emit snapshot at configurable intervals (default: every trade)
    - NOTE: Book reconstruction from trades is approximate. Log a warning that this produces estimated snapshots.
    - If download fails for a date, log warning and continue with available dates.
    - Save reconstructed snapshots to Parquet using `write_lob_parquet`.
    - Return output path.

    **Alternative simpler path (preferred for initial implementation):**
    If book reconstruction from trades is too complex, implement `download_historical` as:
    - Download trade CSVs from the archive
    - For each trade, create a partial snapshot with trade_price, trade_size, trade_side populated
    - Mark order book columns as NaN with a log warning that historical archives only provide trade data
    - The WebSocket recorder (Task 2) is the primary source for full LOB data

    Add `websockets>=12.0` to pyproject.toml dependencies.

    **Error handling:**
    - Retry REST requests up to 3 times with exponential backoff
    - Raise clear errors for network failures, invalid symbols
    - Respect rate limits (Bybit: 120 req/min for public endpoints)
  </action>

<task type="auto">
  <name>Task 2: Implement WebSocket LOB recorder</name>
  <files>lob_forge/data/downloader.py</files>
  <action>
    Add WebSocket recording method to BybitDownloader:

    **Method 3: `record_websocket(duration_seconds: int, output_path: Path | None = None) -> Path`**
    - Connect to `wss://stream.bybit.com/v5/public/linear`
    - Send subscription: `{"op": "subscribe", "args": ["orderbook.{depth}.{symbol}"]}`
    - Handle two message types:
      a. **Snapshot** (`type: "snapshot"`): Full book state. Store as current book.
         - `data.b` = bids [[price, size], ...], `data.a` = asks [[price, size], ...]
         - `data.ts` = timestamp (ms)
      b. **Delta** (`type: "delta"`): Incremental update. Apply to current book.
         - `data.b` = bid updates, `data.a` = ask updates
         - size=0 means remove level, otherwise upsert
         - After applying delta, re-sort bids (desc) and asks (asc), take top {depth} levels
    - After each snapshot/delta, construct a row matching LOB_SCHEMA and append to buffer
    - Flush buffer to Parquet every 1000 rows (configurable) using `write_lob_parquet`
    - Run for `duration_seconds`, then close connection and flush remaining buffer
    - Use `asyncio.run()` internally so caller doesn't need async
    - Handle disconnects: reconnect with exponential backoff, re-subscribe
    - Ping/pong heartbeat every 20 seconds to keep connection alive (Bybit requires this)

    **Book state management:**
    - Use sorted dicts or lists for bid/ask levels
    - Bids: sorted by price descending
    - Asks: sorted by price ascending
    - Delta application: match by price level, update size (remove if size=0)

    **Output:** Parquet file at `{output_dir}/{symbol}_{date}_{time}.parquet`

    Use the `websockets` library (async). Wrap the async recording loop in a sync method using `asyncio.run()`.
  </action>
