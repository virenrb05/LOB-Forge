  <action>
    **configs/validation.yaml:**
    ```yaml
    # @package _global_
    validation:
      checkpoint_path: null          # Path to generator checkpoint (required)
      data_path: null                # Path to preprocessed Parquet data (required)
      n_samples: 1000               # Number of synthetic sequences to generate
      seq_len: 100                   # Sequence length
      ddim_steps: 50                # DDIM sampling steps
      output_dir: outputs/validation # Where to save results
      seed: 42
      run_stylized_facts: true
      run_lob_bench: true
      run_regime_validation: true
      discriminator:
        hidden_dim: 128
        epochs: 50
        lr: 0.001
    ```

    **lob_forge/evaluation/validate_generator.py:**

    `validate_generator(cfg: DictConfig) -> dict[str, Any]`:
    1. Set seed (torch, numpy)
    2. Resolve device (MPS > CUDA > CPU) — same pattern as train.py
    3. Load generator from checkpoint:
       - Instantiate DiffusionModel from cfg.generator
       - Load state dict from checkpoint (use EMA weights if available: checkpoint["ema_state_dict"])
       - Move to device, set eval mode
    4. Load real data:
       - Read preprocessed Parquet from cfg.validation.data_path
       - Extract book columns (40 LOB features) and mid-price series
       - Group by regime labels using LOBSequenceDataset regime computation
    5. Generate synthetic data:
       - For each regime (0, 1, 2): generate n_samples // 3 sequences using model.generate()
       - Concatenate and flatten for metric computation
    6. Run validation suites:
       - If run_stylized_facts: call run_all_stylized_tests → get results + call summary_figure
       - If run_lob_bench: call run_lob_bench with regime labels
       - If run_regime_validation: call validate_regime_conditioning
    7. Combine results into single dict
    8. Save results as JSON to output_dir/validation_results.json
    9. Save summary figure to output_dir/stylized_facts.png (if generated)
    10. Log summary metrics to logger
    11. Return combined results dict

    Add `if __name__ == "__main__":` block using Hydra decorator for CLI usage:
    `python -m lob_forge.evaluation.validate_generator checkpoint_path=... data_path=...`

    Use logging module. Handle missing checkpoint/data gracefully with clear error messages.
