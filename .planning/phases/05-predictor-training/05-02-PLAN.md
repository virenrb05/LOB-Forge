  <name>Task 2: Implement walk-forward rolling-window evaluation</name>
  <files>lob_forge/predictor/walk_forward.py, lob_forge/predictor/__init__.py</files>
  <action>
    Create `lob_forge/predictor/walk_forward.py`:

    **`walk_forward_eval(cfg, data_path, output_dir) -> dict`**

    Walk-forward evaluation trains on window i and evaluates on window i+1,
    using rolling temporal splits. This is the proper way to evaluate time-series
    models — no shuffling, strict temporal ordering, purge gaps between windows.

    Implementation:
    1. Load full dataset length (read Parquet row count without loading all data)
    2. Compute window boundaries:
       - n_windows = cfg.predictor.walk_forward.n_windows (default 3, minimum 2)
       - train_ratio = cfg.predictor.walk_forward.train_ratio (default 0.7)
       - purge_gap = cfg.predictor.walk_forward.purge_gap (default 10)
       - For n_windows evaluation windows, we need n_windows+1 consecutive segments
       - Total data split into (n_windows + 1) equal chunks
       - Window i: train on chunks [0..i], evaluate on chunk [i+1]
       - BUT simpler approach: expanding window
         - Each window = (train_start, train_end, val_start, val_end)
         - Window boundaries with purge gaps between train and val
    3. For each window:
       a. Write temporary train/val Parquet files by slicing source data by row indices
       b. Call train_model(cfg, tmp_train_path, tmp_val_path, window_output_dir)
       c. Collect metrics with window index prefix
       d. Log per-window metrics to wandb if enabled
    4. After all windows:
       - Compute mean and std of key metrics across windows
       - Log aggregated metrics to wandb with "wf_mean/" and "wf_std/" prefixes
       - Return dict with per-window and aggregated metrics
    5. Clean up temporary files

    **Expanding window strategy (simpler, standard in finance):**
    - Given N total rows, split into K+1 equal segments (K = n_windows)
    - Window 1: train on segment 0, val on segment 1
    - Window 2: train on segments 0-1, val on segment 2
    - Window 3: train on segments 0-2, val on segment 3
    - Each has purge_gap rows removed between train end and val start

    Use pyarrow.parquet to read/slice efficiently without loading full DataFrame.

    **Update `lob_forge/predictor/__init__.py`**: Add walk_forward_eval to exports.
  </action>
