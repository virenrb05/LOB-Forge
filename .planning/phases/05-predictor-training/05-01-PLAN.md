  <action>
    Create `lob_forge/predictor/metrics.py` with two main functions:

    **`compute_classification_metrics(y_true, y_pred, n_horizons, n_classes=3)`**
    - Input: y_true (N, n_horizons) int64, y_pred (N, n_horizons) int64
    - Use sklearn.metrics: f1_score, precision_score, recall_score, confusion_matrix
    - Compute PER-HORIZON metrics: for each horizon h, compute per-class F1/precision/recall
    - Compute weighted F1 and macro F1 per horizon
    - Return dict with structure:
      ```
      {
        "horizon_{h}_f1_weighted": float,
        "horizon_{h}_f1_macro": float,
        "horizon_{h}_precision_macro": float,
        "horizon_{h}_recall_macro": float,
        "horizon_{h}_f1_class_{c}": float,  # per class
        "f1_weighted_mean": float,  # mean across horizons
        "f1_macro_mean": float,
      }
      ```
    - Use `zero_division=0` in sklearn calls to handle classes with no predictions

    **`compute_vpin_metrics(y_true, y_pred)`**
    - Input: y_true (N,) float32, y_pred (N,) float32
    - Compute MSE, MAE, and Pearson correlation (numpy)
    - Return dict: {"vpin_mse": float, "vpin_mae": float, "vpin_corr": float}

    Add `scikit-learn` to pyproject.toml dependencies if not already present.

    **Step A: Extend LOBDataset for VPIN targets**

    Edit `lob_forge/data/dataset.py` — add optional `vpin_col: str | None = None` parameter to LOBDataset.__init__:
    - When vpin_col is set, also load that column as float32 array
    - __getitem__ returns `(features, labels, vpin_target)` when vpin_col is set
    - __getitem__ returns `(features, labels)` when vpin_col is None (backward compatible)
    - vpin_target is the scalar value at the END of the window (same index as labels)

    **Step B: Create `lob_forge/predictor/trainer.py`**

    **CRITICAL implementation details:**

    1. **Feature columns**: Models expect exactly 40 book columns in specific order:
       `BID_PRICE_COLS + BID_SIZE_COLS + ASK_PRICE_COLS + ASK_SIZE_COLS` from schema.py.
       LOBDataset auto-detection would pick up ~65 columns (includes derived features).
       The trainer MUST pass explicit `feature_cols` to LOBDataset using these 40 schema constants.

    2. **Model interface**: DualAttentionTransformer.forward() returns `dict[str, Tensor]`
       with keys "logits", "embedding", optional "vpin". DeepLOB and LinearBaseline return
       plain `Tensor` (just logits). Handle both: if output is dict → use output["logits"];
       if Tensor → use directly.

    3. **VPIN loss**: Only compute when model is DualAttentionTransformer with vpin_head=True
       AND model output contains "vpin" key. Use MSELoss weighted by cfg.predictor.vpin_loss_weight (0.1).
       Total loss = focal_loss + vpin_loss_weight * vpin_mse_loss.

    **Functions to implement:**

    **`build_model(cfg) -> nn.Module`**
    - Model factory: cfg.predictor.model selects class
      - "dual_attention" → DualAttentionTransformer(n_levels, features_per_level, d_model, n_heads, ...)
      - "deeplob" → DeepLOB(n_levels, features_per_level, n_classes, n_horizons, lstm_hidden=64)
      - "linear" → LinearBaseline(n_levels, features_per_level, n_classes, n_horizons)
    - Pass config values from cfg.predictor.* to constructor kwargs
    - Return model on correct device

    **`resolve_device(cfg) -> torch.device`**
    - cfg.project.device == "auto": try MPS → CUDA → CPU
    - Otherwise use specified device string

    **`train_model(cfg, train_path, val_path, output_dir) -> dict`**
    - Main training function. Steps:
      1. Resolve device, set seed (cfg.project.seed), set torch deterministic mode
      2. Build model via build_model(cfg)
      3. Create LOBDataset for train/val with explicit feature_cols (40 book columns)
         and vpin_col="vpin_50" when model has VPIN head
      4. Create DataLoaders (batch_size from cfg.training.batch_size, num_workers from cfg.training.num_workers, pin_memory=False for MPS)
      5. Create FocalLoss with gamma=cfg.predictor.focal_loss_gamma (class weights computed from train label distribution)
      6. Create optimizer: AdamW(lr=cfg.predictor.optimizer.lr, weight_decay=cfg.predictor.optimizer.weight_decay)
      7. Create scheduler: OneCycleLR(max_lr=cfg.predictor.scheduler.max_lr, pct_start=cfg.predictor.scheduler.pct_start, epochs=cfg.training.epochs, steps_per_epoch=ceil(len(train_ds)/batch_size/gradient_accumulation))
      8. wandb.init(project=cfg.wandb.project, config=OmegaConf.to_container(cfg)) if cfg.wandb.enabled else None
      9. Training loop:
         - For each epoch:
           a. train_epoch: iterate train loader, forward pass, compute loss (focal + vpin), backward, gradient accumulation (cfg.training.gradient_accumulation steps), optimizer.step(), scheduler.step()
           b. val_epoch: iterate val loader, compute val loss, collect predictions
           c. Compute metrics via compute_classification_metrics + compute_vpin_metrics
           d. Log to wandb: train_loss, val_loss, lr, all metrics
           e. Early stopping: track best val_loss, patience=cfg.training.early_stopping_patience
           f. Checkpoint: save best model state_dict to output_dir/best_model.pt
      10. wandb.finish() if enabled
      11. Return dict with best_metrics, best_epoch, model_path

    **Device handling notes for MPS:**
    - pin_memory=False (MPS doesn't support pinned memory)
    - num_workers=0 (from config, MPS works best with 0)
    - Some operations may need .float() cast (MPS prefers float32)

    **Step C: Update __init__.py**
    Add `build_model` and `train_model` to predictor __init__.py exports.

    **Step D: Add wandb + scikit-learn to pyproject.toml**
    Add `wandb` and `scikit-learn` to project dependencies if not present.
  </action>
