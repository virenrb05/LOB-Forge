<task type="auto">
  <name>Task 1: Unit tests for metrics module</name>
  <files>tests/test_metrics.py</files>
  <action>
    Create `tests/test_metrics.py` following existing test patterns (class-based pytest, see test_predictor.py).

    **TestClassificationMetrics class:**
    - `test_perfect_predictions`: All correct → F1=1.0 for all classes/horizons
    - `test_all_wrong`: All wrong → low F1 values
    - `test_single_horizon`: Works with n_horizons=1
    - `test_multi_horizon`: Works with n_horizons=4, returns per-horizon keys
    - `test_class_imbalance`: Majority class predictions → weighted F1 != macro F1
    - `test_output_keys`: Verify all expected keys present in returned dict
    - `test_known_values`: Hand-computed example:
      y_true = [0,0,1,1,2,2], y_pred = [0,1,1,1,2,0]
      Verify F1 values match sklearn.metrics.f1_score directly

    **TestVpinMetrics class:**
    - `test_perfect_predictions`: y_true == y_pred → MSE=0, MAE=0, corr=1.0
    - `test_known_mse`: Hand-computed MSE for simple arrays
    - `test_correlation`: Linearly related → corr ≈ 1.0
    - `test_output_keys`: Verify vpin_mse, vpin_mae, vpin_corr keys

    Use numpy for creating test arrays. Keep tests fast (no GPU, no data loading).
  </action>
  <verify>python -m pytest tests/test_metrics.py -v</verify>
  <done>All metrics tests pass, covering perfect/imperfect/edge cases</done>
</task>

<task type="auto">
  <name>Task 2: Integration tests for training loop</name>
  <files>tests/test_trainer.py</files>
  <action>
    Create `tests/test_trainer.py` with integration tests that exercise the full training pipeline on tiny synthetic data.

    **Helper: `_create_synthetic_parquet(tmp_path, n_rows=500)`**
    - Generate random DataFrame with all 40 book columns (bid_price_1..10, bid_size_1..10, ask_price_1..10, ask_size_1..10) as float64
    - Add label columns: label_h10, label_h20, label_h50, label_h100 as float64 with values in {0.0, 1.0, 2.0}
    - Add vpin_50 column as float64 in [0, 1]
    - Add timestamp column as int64 (monotonic)
    - Add mid_price, spread, trade_price, trade_size as float64 (random)
    - Add trade_side as int8
    - Write to tmp_path / "train.parquet" and tmp_path / "val.parquet" (can reuse same data for test)
    - Return (train_path, val_path)

    **Helper: `_make_test_cfg(tmp_path)`**
    - Create OmegaConf DictConfig mimicking the real config structure:
      ```python
      cfg = OmegaConf.create({
          "project": {"name": "test", "seed": 42, "device": "cpu"},
          "training": {"epochs": 2, "batch_size": 32, "gradient_accumulation": 1,
                       "early_stopping_patience": 5, "num_workers": 0},
          "predictor": {"model": "dual_attention", "d_model": 16, "n_heads": 2,
                        "n_spatial_layers": 1, "n_temporal_layers": 1,
                        "feedforward_dim": 32, "dropout": 0.0, "n_levels": 10,
                        "features_per_level": 4, "n_classes": 3, "n_horizons": 4,
                        "max_seq_len": 64, "focal_loss_gamma": 2.0,
                        "vpin_head": True, "vpin_loss_weight": 0.1,
                        "optimizer": {"lr": 1e-3, "weight_decay": 0},
                        "scheduler": {"max_lr": 1e-2, "pct_start": 0.3}},
          "data": {"sequence_length": 20, "horizons": [10, 20, 50, 100],
                   "data_dir": str(tmp_path)},
          "wandb": {"project": "test", "enabled": False},
      })
      ```
    - Use small dimensions (d_model=16, n_heads=2) for fast tests
    - wandb disabled in tests
    - device="cpu" for CI portability

    **TestBuildModel class:**
    - `test_build_dual_attention`: cfg.predictor.model="dual_attention" → DualAttentionTransformer instance
    - `test_build_deeplob`: cfg.predictor.model="deeplob" → DeepLOB instance
    - `test_build_linear`: cfg.predictor.model="linear" → LinearBaseline instance
    - `test_build_unknown_raises`: cfg.predictor.model="unknown" → ValueError

    **TestTrainModel class:**
    - `test_train_dual_attention_2_epochs`: Train TLOB for 2 epochs on synthetic data.
      Verify: returns dict with "best_metrics" key, checkpoint file exists.
    - `test_train_deeplob_2_epochs`: Train DeepLOB for 2 epochs on synthetic data.
      Verify: returns dict, no errors.
    - `test_train_linear_2_epochs`: Train LinearBaseline for 2 epochs.
    - `test_vpin_loss_only_for_tlob`: TLOB metrics include vpin_*, DeepLOB metrics don't.
    - `test_early_stopping`: Set patience=1, verify training stops before max epochs
      when val loss doesn't improve (use high lr to cause divergence or just check logic).
    - `test_checkpoint_saved`: After training, best_model.pt exists in output_dir.

    **TestLOBDatasetVpin class:**
    - `test_dataset_without_vpin`: LOBDataset returns 2-tuple (features, labels)
    - `test_dataset_with_vpin`: LOBDataset with vpin_col="vpin_50" returns 3-tuple

    Use `tmp_path` fixture for all file I/O. Tests must run on CPU, no MPS required.
    Keep n_rows small (500) and epochs small (2) for fast execution.
  </action>
  <verify>python -m pytest tests/test_trainer.py -v --timeout=120</verify>
  <done>

    - Training runs 2 epochs on synthetic data without errors
    - Checkpoint files created

  </done>
