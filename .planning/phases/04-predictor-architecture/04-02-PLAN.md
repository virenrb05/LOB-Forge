<task type="auto">
  <name>Task 1: Implement DeepLOB baseline</name>
  <files>lob_forge/predictor/deeplob.py</files>
  <action>
    Create `DeepLOB(nn.Module)` following the simplified architecture from the tech spec (inspired by Zhang et al. 2019).

    Constructor parameters:
    - `n_levels: int = 10` — LOB levels
    - `features_per_level: int = 4` — features per level (bid_price, bid_size, ask_price, ask_size)
    - `n_classes: int = 3` — output classes (DOWN, STATIONARY, UP)
    - `n_horizons: int = 4` — number of prediction horizons (separate head per horizon)
    - `lstm_hidden: int = 64`

    Architecture (input shape: (batch, T, n_levels * features_per_level)):
    1. Reshape input to (batch, 1, T, 40) for Conv2d processing
    2. Conv block 1: Conv2d(1, 16, kernel_size=(1, 2), stride=(1, 2)), LeakyReLU(0.01), BatchNorm2d(16)
       - Output: (batch, 16, T, 20)
    3. Conv block 2: Conv2d(16, 32, kernel_size=(1, 2), stride=(1, 2)), LeakyReLU(0.01), BatchNorm2d(32)
       - Output: (batch, 32, T, 10)
    4. Inception module: three parallel branches on the spatial (last) dimension:
       - Branch A: Conv2d(32, 32, kernel_size=(1, 1))
       - Branch B: Conv2d(32, 32, kernel_size=(3, 1), padding=(1, 0))
       - Branch C: Conv2d(32, 32, kernel_size=(5, 1), padding=(2, 0))
       - Concatenate along channel dim → (batch, 96, T, 10)
    5. Collapse spatial dim: Conv2d(96, 32, kernel_size=(1, 10)) → (batch, 32, T, 1)
    6. Squeeze and permute to (batch, T, 32) for LSTM
    7. LSTM(input_size=32, hidden_size=64, num_layers=1, batch_first=True)
    8. Per-horizon classification heads: nn.ModuleList of n_horizons × nn.Linear(64, n_classes)
       - Each head takes the LAST time step output: lstm_out[:, -1, :]

    Forward returns: (batch, n_horizons, n_classes) logits tensor

    Important: use nn.ModuleList for the per-horizon heads, not a Python list.
  </action>
  <verify>
    ```python
    python -c "
    import torch
    from lob_forge.predictor.deeplob import DeepLOB
    m = DeepLOB(n_levels=10, features_per_level=4, n_classes=3, n_horizons=4)
    x = torch.randn(16, 100, 40)
    y = m(x)
    assert y.shape == (16, 4, 3), f'Expected (16,4,3), got {y.shape}'
    print(f'DeepLOB: OK — {sum(p.numel() for p in m.parameters())} params')
    "
    ```
  </verify>
  <done>DeepLOB produces (batch, n_horizons, 3) from (batch, T, 40). Architecture: 2 Conv blocks → Inception → spatial collapse → LSTM → per-horizon linear heads.</done>
</task>

<task type="auto">
  <name>Task 2: Implement LinearBaseline</name>
  <files>lob_forge/predictor/linear_baseline.py</files>
  <action>
    Create `LinearBaseline(nn.Module)` as a simple logistic regression baseline (performance floor).

    Constructor parameters:
    - `n_levels: int = 10`
    - `features_per_level: int = 4`
    - `n_classes: int = 3`
    - `n_horizons: int = 4`

    Architecture (input shape: (batch, T, n_levels * features_per_level)):
    1. Take only the LAST time step: x[:, -1, :] → (batch, 40)
    2. Per-horizon classification heads: nn.ModuleList of n_horizons × nn.Linear(40, n_classes)

    Forward returns: (batch, n_horizons, n_classes) logits tensor

    This is intentionally simple — it's the performance floor. If TLOB doesn't beat this, something is wrong.

    Note: Do NOT add hidden layers, dropout, or activation — this is a pure linear/logistic baseline. The name "linear" means linear model, not "neural network with linear layers."
  </action>
  <verify>
    ```python
    python -c "
    import torch
    from lob_forge.predictor.linear_baseline import LinearBaseline
    m = LinearBaseline(n_levels=10, features_per_level=4, n_classes=3, n_horizons=4)
    x = torch.randn(16, 100, 40)
    y = m(x)
    assert y.shape == (16, 4, 3), f'Expected (16,4,3), got {y.shape}'
    print(f'LinearBaseline: OK — {sum(p.numel() for p in m.parameters())} params')
    "
    ```
  </verify>
  <done>LinearBaseline produces (batch, n_horizons, 3) from (batch, T, 40). Pure logistic regression on last time step — no hidden layers.</done>
