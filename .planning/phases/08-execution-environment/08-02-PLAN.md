  <action>
Implement `LOBExecutionEnv(gymnasium.Env)` with:

**Constructor params:**
- `lob_data: np.ndarray` — shape (N, 40), the full LOB sequence (40 book columns only, z-score normalized)
- `seq_len: int = 100` — number of LOB snapshots in each observation window
- `inventory: float = 1000.0` — total shares/units to execute (liquidation task)
- `horizon: int = 500` — max steps before episode truncates
- `order_sizes: tuple[float, float, float] = (0.01, 0.05, 0.20)` — market order sizes as fraction of inventory
- `limit_offsets_bps: tuple[float, float, float] = (1, 5, 20)` — limit order price offsets in bps
- `cost_model: CostModel | None = None` — defaults to CostModel() if None

**Gymnasium spaces:**
- `observation_space = gymnasium.spaces.Box(low=-np.inf, high=np.inf, shape=(seq_len, 40), dtype=np.float32)` — sliding window of LOB snapshots
- `action_space = gymnasium.spaces.Discrete(7)` — 7 discrete actions

**Action mapping (store as class constant `ACTION_NAMES`):**
- 0: WAIT — no execution this step, zero cost
- 1: MARKET_SMALL — execute order_sizes[0] * remaining_inventory at best ask
- 2: MARKET_MED — execute order_sizes[1] * remaining_inventory at best ask
- 3: MARKET_LARGE — execute order_sizes[2] * remaining_inventory at best ask
- 4: LIMIT_AGGRESSIVE — place limit at ask - limit_offsets_bps[0] bps; assume 50% fill probability
- 5: LIMIT_MID — place limit at ask - limit_offsets_bps[1] bps; assume 30% fill probability
- 6: LIMIT_PASSIVE — place limit at ask - limit_offsets_bps[2] bps; assume 10% fill probability

**State (reset on episode start):**
- `self._step: int = 0` — current step index into lob_data
- `self._remaining: float` — shares/units left to execute (starts at inventory)
- `self._executed_vwap: float` — running VWAP of executed trades
- `self._episode_cost: float` — cumulative cost for this episode

**reset(seed=None, options=None):**
- Super call for seeding
- Start at random position in lob_data leaving horizon+seq_len steps (so episode fits)
- Reset all state variables
- Return (self._get_obs(), {})

**step(action):**
1. Get current LOB snapshot at self._step: raw_lob = self.lob_data[self._step]
2. Extract mid_price = (col_0 + col_20) / 2 (ask_1 + bid_1, but data is z-score normalized — use raw_lob[0] as ask_price proxy and raw_lob[20] as bid_price proxy)
3. Extract spread = raw_lob[0] - raw_lob[20] (ask_1 - bid_1; may be small after normalization — floor at 0.0)
4. Compute avg_daily_volume = 1e6 (constant proxy — real calibration is Phase 9+ concern)
5. For market actions (1-3): compute exec_size = order_sizes[i] * self._remaining; simulate fill at ask (raw_lob[0]); call cost_model.compute(exec_price=raw_lob[0], exec_size=exec_size, mid_price=mid_price, spread=max(spread, 0.0), avg_daily_volume=avg_daily_volume)
6. For limit actions (4-6): sample fill from Bernoulli(fill_prob); if filled, exec_price = raw_lob[0] - offset_bps * 1e-4 * abs(raw_lob[0]); compute cost at reduced price; if not filled, zero cost
7. For WAIT: zero execution, zero cost
8. Update self._remaining, self._executed_vwap (running VWAP), self._episode_cost
9. Compute reward = -cost / self.inventory (normalized by inventory so reward scale is consistent)
10. Advance self._step += 1
11. terminated = self._remaining <= 1e-6 (fully executed)
12. truncated = self._step >= self._start + self.horizon
13. info = {"remaining": self._remaining, "step": self._step, "episode_cost": self._episode_cost}
14. Return (self._get_obs(), reward, terminated, truncated, info)

**_get_obs():** return self.lob_data[self._step - self.seq_len : self._step].astype(np.float32) — sliding window; pad with zeros at episode start if not enough history

**Class-level constant:**
```python
ACTION_NAMES = ["WAIT", "MARKET_SMALL", "MARKET_MED", "MARKET_LARGE",
                "LIMIT_AGGRESSIVE", "LIMIT_MID", "LIMIT_PASSIVE"]
```

Do NOT use any torch in environment.py — keep it pure numpy for speed.
Do NOT implement render() — return None (not needed for this project).
