<feature>
  <name>CostModel — spread + fee + market impact</name>
  <files>lob_forge/executor/cost_model.py, tests/test_cost_model.py</files>
  <behavior>
    CostModel(fee_bps=2.0, impact_eta=0.1) accepts constructor params.

    compute(exec_price, exec_size, mid_price, spread, avg_daily_volume) -> float:
      - spread_cost = 0.5 * spread * exec_size  (half-spread model)
      - fee_cost = fee_bps * 1e-4 * exec_price * exec_size
      - participation_rate = exec_size / avg_daily_volume
      - impact_cost = impact_eta * mid_price * sqrt(participation_rate) * exec_size
      - total = spread_cost + fee_cost + impact_cost  (always >= 0)

    Cases:
      - (exec_price=100.0, exec_size=1.0, mid_price=100.0, spread=0.01, avg_daily_volume=1e6):
          spread_cost = 0.005, fee_cost = 0.0002, impact_cost ≈ 0.0001
          total ≈ 0.0053  (small for tiny order)
      - exec_size=0 → total = 0.0
      - Zero spread → spread_cost = 0, other components still apply
      - Large order (exec_size=1e4, avg_daily_volume=1e6) → high impact_cost dominates
      - Negative exec_size not valid (raise ValueError)
  </behavior>
  <implementation>
    Define CostModel as a dataclass or plain class with fee_bps and impact_eta as constructor params.
    compute() takes (exec_price, exec_size, mid_price, spread, avg_daily_volume) as floats.
    Use math.sqrt for impact term — no torch dependency needed.
    Validate exec_size >= 0 (raise ValueError if negative).
    All arithmetic uses float throughout.
  </implementation>
</feature>
