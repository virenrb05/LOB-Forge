  <files>lob_forge/executor/environment.py</files>
  <action>
Extend LOBExecutionEnv to support synthetic data mode. Modify constructor and reset():

**Constructor changes:**
Add these new params (after existing params):
- `mode: str = "real"` — `"real"` or `"synthetic"`
- `generator: DiffusionModel | None = None` — required when mode="synthetic"; must not be None
- `regime: int = 1` — volatility regime for synthetic generation (0=low-vol, 1=normal, 2=high-vol)
- `device: str = "cpu"` — device for generator inference

Validate: if mode="synthetic" and generator is None, raise ValueError("generator required for synthetic mode")

**Import guard (avoid circular/heavy import at module level):**
Use TYPE_CHECKING guard for DiffusionModel type annotation:
```python
from __future__ import annotations
from typing import TYPE_CHECKING
if TYPE_CHECKING:
    from lob_forge.generator.model import DiffusionModel
```
At runtime, accept Any for generator param — duck-type it (call generator.generate() directly).

**reset() changes for synthetic mode:**
When mode="synthetic":
1. Import torch (lazy import inside method: `import torch`)
2. Generate one episode's worth of data:
   ```python
   n_steps = self.horizon + self.seq_len + 10  # buffer
   regime_t = torch.tensor([self.regime] * 1)  # batch of 1
   with torch.no_grad():
       generated = self.generator.generate(
           n_samples=1,
           seq_len=n_steps,
           regime=regime_t.to(self.device),
           method="ddim",
       )  # shape (1, n_steps, 40)
   self.lob_data = generated[0].cpu().numpy().astype(np.float32)  # (n_steps, 40)
   ```
3. Set self._start = self.seq_len (always start from index seq_len for synthetic, no randomization needed)
4. Reset all state variables (same as real mode reset)

For real mode, reset() stays unchanged from Plan 08-02.

Do NOT store a permanent reference to generated data between episodes — regenerate on every reset() in synthetic mode (allows regime variation per episode).
